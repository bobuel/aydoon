# Alex Aidun — AI Product Portfolio

This repository powers my personal AI product portfolio: a curated view of the products, prototypes, and systems I am building across education, knowledge work, creative tools, and agentic workflows.

The purpose of the portfolio is not to show every experiment. It is to show a coherent product practice: spotting messy human workflows, turning them into usable interfaces, and applying AI where it can make people calmer, faster, and more capable.

## Positioning

I am an AI product leader with a builder's instincts. My work sits at the intersection of product strategy, education, documentation, agentic systems, and hands-on prototyping.

The common thread across these projects is practical AI: tools that help people understand information, make better decisions, extend expert workflows, or create something they could not easily create before.

## Featured work

The portfolio currently highlights several product directions:

- **KidGrow** — connects school-day information to family-day learning through parent-friendly summaries and simple do-together activities.
- **CertifyFast** — helps teams turn source material into defensible certification assets, including job task analysis, exam questions, evidence mapping, and review workflows.
- **BloomGPT / Bloom's Quiz Builder** — supports teachers in generating Bloom's Taxonomy-aligned questions from educational materials.
- **Habari / Informa-style briefings** — explores higher-signal news and research synthesis with de-duplication, source comparison, and editorial judgment.
- **Creative AI prototypes** — experiments with AI-assisted games, storytelling, multimodal interaction, and playful product surfaces.

## Product principles

Across the portfolio, I try to build around a few principles:

1. **AI should reduce friction, not create more work.**
2. **Expert workflows need structure, evidence, and review — not just generation.**
3. **The best AI products feel specific to the user's real context.**
4. **Trust comes from clear boundaries, not overclaiming.**
5. **Prototypes are useful when they teach something concrete.**

## How the portfolio is built

This is a React / TypeScript app. It includes a portfolio view and an optional AI assistant experience that can answer questions about the projects listed in the app.

The assistant is intentionally constrained to the portfolio content. It is meant as a lightweight interface for exploring the work, not as a general-purpose chatbot.

## Local development

**Prerequisites:** Node.js

```bash
npm install
npm run dev
```

If using the AI assistant locally, set the required Gemini API key in your local environment file.

## Status

This portfolio is an active work in progress. The public presentation is being refined to emphasize the strongest product stories, case studies, and lessons learned rather than every prototype or tool experiment.
