# AGENTS.md

Guidance for AI coding agents working in this repository.

## Product intent

This is Alex Aidun's AI product portfolio. The goal is to present a coherent product practice, not a raw archive of experiments.

The portfolio should position Alex as an AI product leader who builds practical agentic systems for education, expert workflows, signal synthesis, and creative interaction.

## Voice and positioning

Use language that emphasizes:

- product judgment
- user workflows
- evidence-grounded generation
- practical AI
- education and assessment expertise
- trusted, useful systems
- creative but disciplined prototyping

Avoid language that over-emphasizes:

- tool novelty
- vibe coding
- generic AI hype
- platform-first framing such as “built with Base44” or “built with AI Studio”
- unsupported claims

## Content rules

- Keep project descriptions outcome-focused.
- Explain who each product is for and what workflow it improves.
- Do not invent metrics, users, customers, partnerships, or external validation.
- Treat private projects as case studies unless explicit public links exist.
- Keep the assistant constrained to the portfolio content.
- Prefer clear, confident, modest writing over grandiose language.

## Technical notes

- This is a React / TypeScript portfolio app.
- Do not commit API keys or local environment files.
- The Gemini assistant should rely on portfolio constants and avoid hallucinating projects not listed.
- If adding new projects, update both the portfolio display and any assistant context that depends on project data.

## Definition of done

A change is good when it makes Alex look more coherent, credible, and product-minded without making unsupported claims.
